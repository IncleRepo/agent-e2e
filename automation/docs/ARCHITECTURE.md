# 구조 원칙

## Core

실행, 인증 상태 보관, 진단, 읽기 전용 차단, 보고서처럼 모든 프로젝트에서 동일한 기능을 담당한다. 제품명, 메뉴명, 고유 API 경로를 포함하지 않는다.

## Project profile

프로젝트 URL, 인증 선택자, 라우트, 알려진 이슈, 실제 사용자 워크플로를 담당한다. 한 프로젝트의 변경이 다른 프로젝트 시나리오에 영향을 주지 않아야 한다.

## Agent contract

Markdown은 AI가 작업 맥락과 절차를 이해하도록 한다. JSON 프로필 검증과 TypeScript 검사는 자연어 규칙이 실행마다 달라지는 것을 방지한다. AI는 유지보수자이며 실제 판정은 Playwright와 검증 코드가 수행한다.
