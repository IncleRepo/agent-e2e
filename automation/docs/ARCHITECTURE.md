# 구조를 나누는 기준

## 공통 엔진

`automation/core/`는 실행, 로그인 상태 보관, 브라우저 진단, 쓰기 요청 차단, 보고서 생성을 맡는다. 서비스 이름이나 특정 메뉴, 고유 API 경로는 이곳에 넣지 않는다.

## 프로젝트 프로필

`automation/projects/<project-id>/`에는 한 서비스에만 필요한 내용을 둔다. URL, 로그인 선택자, 화면 경로, 알려진 이슈, 실제 사용자 흐름이 여기에 해당한다.

한 프로젝트의 변경이 다른 프로젝트 테스트에 영향을 주지 않는 구조가 기준이다. 같은 구현이 두 프로젝트 이상에서 반복되고 동작도 같을 때만 공통 엔진으로 옮긴다.

## AI와 실행 코드의 역할

Markdown 문서는 AI 에이전트가 작업 순서와 판단 기준을 놓치지 않게 돕는다. JSON 프로필 검증과 TypeScript 검사는 그 규칙을 실행 가능한 형태로 고정한다.

AI는 설정과 시나리오를 관리하지만, 최종 통과 여부는 Playwright와 검증 코드가 판단한다.
