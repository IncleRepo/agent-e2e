# 프로젝트 온보딩

## 필요한 입력

- 고유 프로젝트 ID
- FE와 BE 소스 경로
- 실행 중인 FE와 API 주소
- 테스트 계정 환경변수
- 로그인 성공 경로
- 읽기 전용으로 허용할 사용자 행동

## 생성

```powershell
npm --prefix automation run project:init -- --project my-project
```

생성된 `automation/projects/my-project/`만 프로젝트에 맞게 수정한다. 공통 기능이 실제로 두 프로젝트 이상에서 반복될 때만 `core` 승격을 검토한다.

## 조사 순서

1. 라우터와 메뉴 구조
2. 로그인 및 권한 처리
3. 공통 API 클라이언트
4. 목록, 상세, 검색, 필터, 페이지 이동
5. 다운로드
6. 오류 경계와 로딩 UI
7. 실제 API 데이터로 동적 ID를 구하는 방법

## 완료 기준

- 프로필 검증 통과
- 로그인 성공
- 주요 정적 라우트 순회
- 최소 한 개의 핵심 사용자 워크플로
- 쓰기 요청 차단 동작 확인
- 통합 보고서 생성
