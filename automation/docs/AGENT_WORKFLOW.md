# AI 에이전트 작업 흐름

## 신규 프로젝트

1. 루트 `AGENTS.md`와 안전 규칙을 읽는다.
2. `project:init`으로 프로젝트 폴더를 생성한다.
3. FE/BE 소스와 실제 서비스 URL을 읽기 전용으로 조사한다.
4. `project.json`의 URL, 인증 선택자, 오류·로딩 기준을 채운다.
5. 주요 정적 화면을 `routes.json`에 작성한다.
6. 동적 상세 대상은 프로젝트 `support/`에서 실제 API 응답으로 선택한다.
7. 검색, 필터, 페이지, 탭, 다운로드는 `tests/`에 시나리오로 작성한다.
8. `validate`, `typecheck`, `audit` 순서로 실행한다.
9. 보고서에서 오탐과 실제 제품 이슈를 구분한다.

## 기존 프로젝트 변경

1. 변경 기능과 기대 동작을 확인한다.
2. 기존 시나리오가 이미 해당 동작을 검증하는지 확인한다.
3. 내부 리팩터링이면 기존 회귀검사만 실행한다.
4. 사용자 행동이나 계약이 추가됐으면 프로젝트 시나리오를 추가한다.
5. 의도적인 URL/API 계약 변경일 때만 기존 기대값을 수정한다.

## 결과 전달

- 전체 시나리오 수와 성공·실패
- 신규 이슈와 기존 알려진 이슈
- 추가 검토 필요 항목
- 실행하지 못한 범위와 이유
- `reports/report.html` 위치
