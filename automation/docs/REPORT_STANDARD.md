# 보고서 기준

실행 결과는 `reports/report.html` 하나에 모으며, 새로 실행할 때마다 이전 결과를 덮어쓴다. 별도 파일을 찾아보지 않아도 전체 흐름을 파악할 수 있어야 한다.

보고서는 아래 순서로 읽힌다.

1. 이번 실행 요약
2. 메뉴별 시나리오 결과
3. 발견된 이슈
4. 추가 확인이 필요한 항목
5. 서버와 로그인 준비 결과

이슈 행은 `대메뉴 | 메뉴 | 개발 | 수정사항 | 타입 | 우선순위 | 작성자 | 수정자 | 수정여부 | 비고` 형식을 유지한다. HTML 안에서 TSV로 내려받을 수 있어야 한다.

실패한 시나리오의 스크린샷과 Trace만 `reports/evidence/`에 남긴다. 성공한 실행에서 생긴 빈 증거 폴더는 정리한다.

비밀번호, 토큰, Authorization 헤더, 쿠키, 개인정보는 보고서와 증거 파일에 포함하지 않는다.
