# 읽기 전용 안전 규칙

프로젝트 시나리오는 반드시 `core/test`를 사용한다. 이 fixture는 `GET`, `HEAD`, `OPTIONS`를 제외한 네트워크 요청을 기본으로 차단한다.

로그인이나 토큰 갱신에 꼭 필요한 요청이 있다면 사용자 확인 뒤 메서드와 경로를 정확히 허용한다.

```json
{
  "safety": {
    "allowedWrites": [
      { "method": "POST", "pathname": "/api/auth/refresh" }
    ]
  }
}
```

전체 POST 요청이나 넓은 경로 패턴을 한꺼번에 열어 두지 않는다.

아래 동작은 사용자 승인과 전용 테스트 데이터가 모두 준비된 경우에만 다룬다.

- 등록, 수정, 삭제
- 권한과 비밀번호 변경
- 승인이나 상태 변경 같은 수동 조치
- 메시지와 이메일 전송
- 결제와 주문
- 운영 DB 변경

인증정보는 `.env.e2e.local`에만 저장한다. 코드, Git, 로그, 보고서에는 값을 남기지 않는다.
