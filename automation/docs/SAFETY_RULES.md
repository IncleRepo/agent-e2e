# 읽기 전용 안전 규칙

프로젝트 시나리오는 `core/test`를 사용해야 한다. 이 fixture는 기본적으로 `GET`, `HEAD`, `OPTIONS` 외 네트워크 요청을 차단한다.

로그인 이후 토큰 갱신처럼 반드시 필요한 요청만 프로젝트 설정에 정확히 허용한다.

```json
{
  "safety": {
    "allowedWrites": [
      { "method": "POST", "pathname": "/api/auth/refresh" }
    ]
  }
}
```

광범위한 경로 패턴이나 전체 POST 허용은 금지한다.

다음 행동은 사용자 승인과 전용 테스트 데이터 없이는 수행하지 않는다.

- 등록, 수정, 삭제
- 권한과 비밀번호 변경
- 수동 조치
- 메시지와 이메일 전송
- 결제와 주문
- 운영 DB 변경

인증정보는 `.env.e2e.local`에만 두고 Git에 포함하지 않는다.
